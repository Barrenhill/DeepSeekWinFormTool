2. 安装与首次激活
2.1 系统要求
操作系统：Windows 7 / 10 / 11（64位）

运行环境：无需安装任何额外环境，双击 .exe 即可运行

网络要求：需要稳定互联网连接（用于调用DeepSeek API）

DeepSeek账号：需要自行注册 DeepSeek平台 并获取API Key

2.2 首次启动与激活
首次运行软件时，会弹出授权注册窗口：

复制机器码：窗口会自动显示您电脑的唯一机器码，请复制并发送给开发者。

获取注册码：开发者会根据机器码生成对应的注册码并返还给您。

输入注册码：将收到的注册码粘贴到输入框中，点击“激活”。

激活成功：后续使用不再弹出注册窗口。

💡 机器码基于您电脑的硬件生成，重装系统或更换硬盘可能导致机器码变化，需重新激活。

3. 快速上手（3步走）
启动软件：双击 DeepSeekBatchTool.exe，进入主界面。

拖入文件：将您的 .xlsx 或 .txt 文件直接拖入软件窗口任意区域。

一键处理：点击 “🚀 开始批量处理” 按钮，等待完成。结果自动保存在原文件同目录下。

就是这么简单！ 无需任何配置，拖入即用。

6. 详细操作指南
6.1 准备数据文件
Excel文件（.xlsx）：

待处理文本放在第一列（A列），从第1行开始填写

软件自动读取A列所有非空单元格

支持最多约100万行（受Excel和内存限制）

TXT文件（.txt）：

每行一条待处理文本，按行拆分

编码建议使用UTF-8（避免中文乱码）

6.2 自定义系统指令
在主界面的 “系统指令”文本框 中，输入或粘贴您想要的指令。

可以从上方 第5章 的常用指令表中复制。

软件会记住您上次输入的指令（配置文件保留）。

6.3 开始处理
点击 “🚀 开始批量处理” 按钮。

进度条实时更新，日志区域显示处理状态。

缓存命中 时日志显示“缓存命中”，表示该条结果来自缓存，不消耗Token。

6.4 查看与保存结果
Excel文件：原目录生成 原文件名_AI处理结果.xlsx，A列为原文，B列为AI输出。

TXT文件：原目录生成 原文件名_AI结果.txt，按顺序保存所有输出。

主界面日志区会显示前5条结果作为预览。

8. 常见问题与注意事项
Q1：首次启动弹出“授权”窗口怎么办？
这是正常流程。将窗口中的机器码复制发给开发者，收到注册码后粘贴输入即可激活。

Q2：“缓存”是什么？为什么要开启？
缓存是软件自动保存的历史处理结果。相同文本第二次处理时直接从缓存读取，不消耗API费用且响应速度极快。缓存文件 ai_cache.json 保存在软件目录下，请勿随意删除。

Q3：运行时报错“Unauthorized”或“认证失败”？
原因：DeepSeek API Key 无效、已过期或账户余额不足。

解决：联系开发者检查或更新 appsettings.json 中的 API Key，并确认DeepSeek账户有足够余额。

Q4：出现“400 Bad Request”错误？
原因：通常是因为文本中含有特殊字符或编码问题。

解决：尝试将文本内容复制到记事本，另存为UTF-8编码后重新处理。如果仍然报错，请联系开发者。

Q5：处理速度很慢？
影响因素：网络状况、API并发限制、文本长度。

优化建议：检查网络连接，确保API并发数不超过限制（默认3，可在配置中调整）。

Q6：窗口显示不完整或按钮被遮住？
解决：请检查系统的“文本缩放”设置是否为100%（在Windows“显示设置”中调整），或者将窗口分辨率调高。

Q7：缓存文件（ai_cache.json）可以删除吗？
可以。删除后所有缓存失效，下次处理会重新调用API消耗Token。建议保留以节省费用。如需清空缓存，也可以直接删除该文件。

Q8：如何修改API Key或并发数？
打开软件目录下的 appsettings.json 文件（用记事本即可）。

修改 "ApiKey"、"MaxConcurrent" 等字段。

保存后重启软件生效。

Q9：软件可以在多台电脑上使用吗？
每台电脑会生成不同的机器码，需要分别激活。

如有批量采购需求，请联系开发者获取批量授权方案。
